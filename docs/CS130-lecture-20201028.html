Search miss





Worst case running time for 2-3 tree search is O(lgn). Remember the height of a 2-3 tree is between floor(log3N) and floor(log2N). So in the worst case it searches the entire height of the tree + 1.

It inserts it into the 3 node to preserve the balance of the 2-3 tree.

Creates a temporary 4 node, then it splits.



The worst-case running time for 2-3 tree insertion is O(lgn)
